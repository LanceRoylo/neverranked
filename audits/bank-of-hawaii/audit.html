Technical Audit — Bank of Hawaii

Auditor: Never Ranked Sample date: 2026-05-08 Pages sampled: 9 representative pages across homepage, timeline/history, annual report, service pages, blog, team profiles, careers, community news

Summary

Bank of Hawaii’s site is content-rich and visually polished, with proper Open Graph social cards deployed across all 9 sampled pages. The foundation is compromised by missing canonicals (100% of sampled pages), inconsistent H1 structure (3 pages have none, 2 have multiples), and sparse schema deployment outside a few key pages. The site has testimonial and rating language on 3 pages but no structured AggregateRating markup, meaning AI engines have no machine-readable hook to cite customer satisfaction.

Overall grade: C (good content and social layer, serious gaps in canonical discipline and schema coverage)

Findings

1. Make sure Google and AI engines see the right version of each page ⚠ HIGH PRIORITY

Every page in your site is missing a canonical tag (9 of 9 sampled pages). The canonical tag tells Google, ChatGPT, Perplexity, and other crawlers which URL is the “official” version when duplicates or near-duplicates exist (query parameters, tracking codes, www vs non-www, http vs https). Without it, you risk diluted link equity, indexing confusion, and fragmented citation in AI answer engines. Even if you think you have no duplicates, the tag acts as insurance and is a best practice signal.

Impact: Search engines may split your page authority across multiple URLs. AI engines may cite a variant URL instead of your preferred one, or skip the page entirely if they detect ambiguity.

Fix:

  • WordPress (Yoast SEO or Rank Math): Canonical is auto-generated by default. Check Settings → Permalinks and confirm your Preferred Domain is set. Verify the plugin is active and canonical output is enabled in Yoast → Search Appearance → General or Rank Math → Titles & Meta.
  • Squarespace: Canonical tags are auto-added. If missing, check Settings → Advanced → Code Injection to ensure no custom code is stripping them.
  • Webflow: Canonical is auto-generated. Verify in Page Settings → SEO Settings that you haven’t overridden it with a custom value pointing elsewhere.
  • Wix: Canonical tags are automatically added. Confirm in Site SEO settings that your primary domain is correctly configured.
  • Custom-coded or enterprise CMS: Add this to the <head> of every page, with CANONICAL_URL replaced by the live, published URL of that page:
<link rel="canonical" href="https://www.boh.com/CANONICAL_URL">

For your homepage (https://www.boh.com):

<link rel="canonical" href="https://www.boh.com">

For the timeline page (https://www.boh.com/125/timeline):

<link rel="canonical" href="https://www.boh.com/125/timeline">

Technical detail: The rel="canonical" link element must appear in the <head> section, must use an absolute URL (include the https:// and domain), and should point to the URL you want indexed. Google’s documentation: developers.google.com/search/docs/crawling-indexing/consolidate-duplicate-urls.

2. Let AI engines cite your customer proof ⚠ HIGH PRIORITY

We found testimonial language and rating references on 3 of your pages (/125/timeline, /about-us/annual-report-2024/year-in-review, and the same annual report page), but zero AggregateRating schema across the entire site. When ChatGPT, Perplexity, or Google’s AI Overviews evaluate “best bank in Hawaii” or “most trusted Hawaii financial institution,” they look for structured ratings (star count, review count, rating value). Without this markup, they have no machine-readable social proof to cite, even though your prose mentions customer satisfaction.

Impact: AI engines will favor competitors with structured rating data when answering queries about reputation, trust, or “best” comparisons. You lose the credibility lift that comes from being cited with a star count.

Fix (NeverRanked schema deployment):

We will deploy AggregateRating schema via our snippet for the pages where you have rating or testimonial evidence. This does not require you to edit code.

Fix (if you want to add it manually before our deployment):

Identify 2-3 pages where you have verifiable customer ratings (Google Business Profile, Trustpilot, internal NPS data, etc.). Add this JSON-LD block to the <head> or end of <body>, replacing placeholders with your actual figures:

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BankOrCreditUnion",
  "name": "Bank of Hawaii",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.6",
    "reviewCount": "487"
  }
}
</script>

Technical detail: The ratingValue must be on a 1-5 or 1-10 scale and must reflect real customer data (Google’s policies prohibit fabricated ratings). The reviewCount must be an integer. This schema is part of the schema.org Organization vocabulary. Validation tool: validator.schema.org.

3. Fix heading structure on 5 of 9 pages ⚠ MEDIUM PRIORITY

3 pages have no H1 tag at all (/about-us/annual-report-2024/year-in-review, /bank-by-appointment, /community/olelo). 2 pages have multiple H1s (https://www.boh.com with 4, /careers/benefits with 2). The H1 is the primary semantic heading that tells search engines and screen readers what the page is about. Pages without an H1 are harder for crawlers to categorize. Pages with multiple H1s dilute the topical signal.

Impact: Google may struggle to understand the primary topic of pages missing an H1. AI engines may skip these pages when they lack a clear topical anchor. Accessibility suffers (screen reader users navigate by heading hierarchy).

Fix:

Each page should have exactly one H1, and it should be the main headline that describes the page’s purpose.

How to fix:

  • WordPress (block editor): Click the heading block, use the toolbar to change “Heading 1” to “Heading 2” (or vice versa).
  • WordPress (classic editor): Highlight the heading text, use the Format dropdown to select “Heading 1” or “Heading 2”.
  • Squarespace / Webflow / Wix: Click the heading, use the text style dropdown to select “Heading 1” or “Heading 2”.
  • Custom-coded: Change <h2> to <h1> (or <h1> to <h2>) in your HTML template.

Technical detail: HTML5 allows multiple H1s per page, but search engines and accessibility guidelines still favor the single-H1 pattern for clarity. The first H1 should appear early in the DOM and match or closely paraphrase the <title> tag.

**Where to add it:**

Your site appears to be custom-coded (no WordPress, Squarespace, or Webflow meta tags detected). You'll need to paste the JSON-LD block below into the `<head>` section of your sitewide template (likely a header include or layout file that wraps every page).

If you're using a tag manager:

If you're using a CMS we didn't detect:

```json

        "postalCode": "96813",
        "addressCountry": "US"

2. Mark up your expert team so AI engines can cite them by name ⚠ HIGH

You have individual expert bio pages (example: Roger Khlopin, CFA in the Center for Family Business and Entrepreneurs). Right now those pages have no Person schema, so when someone asks “Who are the investment advisors at Bank of Hawaii?” AI engines can’t surface names, credentials, or specialties. Add Person schema to every expert bio page. This also helps Google show rich results in Knowledge Panels and lets Perplexity cite your advisors as authoritative sources when answering finance questions.

Where to add it:

You’ll need to add this schema on a per-page basis (one block per expert). If your site is built with a CMS or component system: - WordPress: Use a schema plugin (Schema Pro, Rank Math Pro) with Person post type. Or paste JSON-LD into the post editor in an HTML block. - Squarespace: No native Person schema. Paste JSON-LD in Page Settings > Advanced > Page Header Code Injection. - Webflow: Page Settings > Custom Code > Head Code. Paste one block per expert page. - Custom-coded: Add the JSON-LD block below to the <head> of each bio page template. Parameterize the fields (name, jobTitle, description, image, email) so you can populate them dynamically from your CMS or database.

Fix — ready-to-paste code (example for Roger Khlopin):

{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Roger Khlopin",
  "honorificSuffix": "CFA",
  "jobTitle": "Senior Vice President, Investment Management",
  "description": "Roger Khlopin advises business owners on investment management strategy to support growth, continuity, and legacy planning.",

  "url": "https://www.boh.com/business/center-for-family-business-and-entrepreneurs/experts/roger-khlopin",
  "worksFor": {
    "@type": "BankOrCreditUnion",
    "@id": "https://www.boh.com/#organization"
  },
  "sameAs": [
    "https://www.linkedin.com/in/rogerkhlopin"
  ],
  "email": "mailto:roger.khlopin@boh.com"
}

Why this specific structure: The worksFor property links back to the sitewide Organization schema via @id reference, so AI engines understand Roger is part of Bank of Hawaii. The honorificSuffix field captures the CFA credential. The sameAs array lets you add LinkedIn, Twitter, or other professional profiles. The email property is optional (remove if you don’t want to publish it). Update the image path, url, and social links for each expert. Repeat this block structure for every expert bio page, changing the field values.

4. Add BreadcrumbList to every non-home page ⚠ MEDIUM

Only 2 of 9 sampled pages have BreadcrumbList schema (the annual report and the bank-by-appointment pages). The other 7 pages (including blog, expert bios, careers, and community pages) have no breadcrumb markup. Google uses BreadcrumbList to understand site hierarchy and display breadcrumb trails in search results. AI engines use it to map your content structure. Add BreadcrumbList schema to every page below the homepage.

Where to add it:

This should be sitewide template logic, not per-page manual work. Your breadcrumb navigation likely already exists in the HTML. You’re just adding the structured data layer.

  • WordPress: Yoast SEO and Rank Math auto-generate BreadcrumbList if breadcrumbs are enabled in your theme. Check SEO plugin settings > Breadcrumbs > Enable.
  • Squarespace: No native breadcrumb schema. You’ll need custom code in Settings > Advanced > Code Injection (Header). Use template logic to populate the breadcrumb items dynamically.
  • Webflow: Add JSON-LD to the page template. Use Webflow’s CMS fields to populate breadcrumb item names and URLs dynamically.
  • Custom-coded: Add the JSON-LD block below to every page template (or use a shared layout component). Parameterize the itemListElement array so it reflects the actual page hierarchy.

Fix — ready-to-paste code (example for expert bio page):

{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://www.boh.com"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Business",
      "item": "https://www.boh.com/business"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "Center for Family Business & Entrepreneurs",
      "item": "https://www.boh.com/business/center-for-family-business-and-entrepreneurs"
    },
    {
      "@type": "ListItem",
      "position": 4,
      "name": "Experts",
      "item": "https://www.boh.com/business/center-for-family-business-and-entrepreneurs/experts"
    },
    {
      "@type": "ListItem",
      "position": 5,
      "name": "Roger Khlopin",
      "item": "https://www.boh.com/business/center-for-family-business-and-entrepreneurs/experts/roger-khlopin"
    }
  ]
}

Why this specific structure: Each ListItem represents one level in the site hierarchy, starting with the homepage (position 1) and ending with the current page. The item property is the URL, and name is the anchor text. The position property must increment by 1 for each level. Update the itemListElement array to match the actual breadcrumb trail on each page. The final item (current page) should not have an item URL if you want to signal it’s the active page, but Google accepts it either way (we’ve included it here for consistency).
