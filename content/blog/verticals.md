# Vertical Taxonomy

Every vertical Never Ranked could realistically target with AEO
content, organized by tier (commercial value to us) and category.

For each vertical: target queries, audience profile, schema gaps
typical of the category, citation surface (where AI engines
currently pull answers from), and post status.

## How to use this taxonomy

When choosing the next post topic:

1. Filter to ⏳ **planned** verticals.
2. Cross-check against `calendar.md` to see what was published in
   the last 60 days. Avoid the same tier twice in a row to keep the
   blog topic mix balanced.
3. Pick the vertical with the highest commercial value to Never
   Ranked right now (Tier 1 first, Tier 6 last) unless there is
   a specific reason to skip the order (a current customer's
   industry, a seasonal hook, a news event).
4. After publishing, update the status to ✅ posted and add the
   live URL.

## Vertical-specific schema gaps cheat sheet

The same five gaps appear across most verticals. Each post should
identify which apply to that specific vertical:

1. **No LocalBusiness or industry-specific schema** (FinancialService,
   Restaurant, MedicalBusiness, LegalService, HomeAndConstructionBusiness, etc.)
2. **No Person schema for the practitioner(s)** with credentials,
   awards, sameAs links to professional registrars
3. **No Service schema for each offering** with provider, areaServed,
   serviceType
4. **No AggregateRating + FAQPage** on the firm's own site (engines
   default to Yelp/Google which the firm does not control)
5. **No owned content the engines can pull from** (no Article schema
   library on long-tail queries, no Reddit presence)

Most posts can lean on this five-gap structure as the spine and
swap the specifics per vertical.
